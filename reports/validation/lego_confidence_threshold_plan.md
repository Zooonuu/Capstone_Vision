# Lego Confidence Threshold Plan

현재는 카메라 연결과 `battery`/`coin` 라벨 이미지가 없으므로, 이 문서는 레고 이미지로만
임시 `confidence_threshold`를 고르는 기준을 정리합니다. 최종 threshold는 `battery`/`coin` 데이터와
실제 카메라 입력이 준비된 뒤 다시 확정해야 합니다.

## 목적

- 레고 이미지에서 너무 낮은 confidence 때문에 오탐이 많이 나오는지 확인합니다.
- threshold를 너무 높였을 때 실제 레고를 놓치는지 확인합니다.
- ROS2 비전 노드의 임시값으로 쓸 threshold 후보를 하나 고릅니다.

## 비교할 값

`robot_ws/src/robot_perception/config/yolo_params.yaml`의 현재 기본값은 다음입니다.

```yaml
confidence_threshold: 0.3
```

우선 아래 네 값을 비교합니다.

| 후보 threshold | 확인 목적 |
|---:|---|
| 0.2 | 낮은 기준에서 레고를 최대한 놓치지 않는지 확인 |
| 0.3 | 현재 기본값 유지 가능성 확인 |
| 0.4 | 오탐을 줄이면서 레고 검출이 유지되는지 확인 |
| 0.5 | 높은 기준에서도 충분히 안정적인지 확인 |

## 테스트 방법

1. 레고 테스트 이미지를 10장 이상 준비합니다.
2. 각 이미지마다 threshold 0.2, 0.3, 0.4, 0.5로 추론합니다.
3. 이미지별로 실제 레고 개수, 검출 개수, 놓친 레고 수, 명확한 오탐 수를 기록합니다.
4. `reason_codes`와 `robot_action`은 카메라/ROS2 연결 뒤 확인하고, 지금은 모델 검출 결과 위주로 비교합니다.

## 임시 선택 기준

레고 이미지 기준으로는 아래 조건을 가장 잘 만족하는 threshold를 임시값으로 둡니다.

1. 실제 레고를 너무 많이 놓치지 않는다.
2. 바닥 무늬, 그림자, 배경 물체를 레고로 잘못 잡는 경우가 적다.
3. 같은 이미지에서 threshold를 조금 바꿔도 결과가 크게 흔들리지 않는다.

최종 발표/시연용 threshold는 레고만으로 확정하지 않습니다. `battery`/`coin` 라벨 이미지가 도착하면
같은 표에 두 class를 추가하고, 실제 카메라 입력까지 연결된 뒤 다시 비교합니다.
